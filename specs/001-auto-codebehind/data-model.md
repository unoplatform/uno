# Data Model: Auto-Generate Code-Behind for XAML Files

**Feature**: 001-auto-codebehind | **Date**: 2026-03-05

## Entities

### 1. XamlFileInfo (Generator Input)

Represents a XAML file candidate for code-behind generation.

| Field | Type | Source | Description |
|-------|------|--------|-------------|
| `FilePath` | `string` | `AdditionalText.Path` | Absolute path to the `.xaml` file |
| `Content` | `SourceText` | `AdditionalText.GetText()` | XAML file content for parsing |
| `SourceItemGroup` | `string` | `build_metadata.AdditionalFiles.SourceItemGroup` | MSBuild item type: `Page` or `ApplicationDefinition` |
| `GenerateCodeBehindOverride` | `string?` | `build_metadata.AdditionalFiles.UnoGenerateCodeBehind` | Per-file override: `"true"`, `"false"`, or `null` (use project default) |

### 2. XamlClassInfo (Parsed Result)

Extracted from XAML content; used to determine if generation is needed.

| Field | Type | Description |
|-------|------|-------------|
| `FullClassName` | `string` | Full `x:Class` value (e.g., `"MyApp.Views.MainPage"`) |
| `Namespace` | `string` | Extracted namespace (e.g., `"MyApp.Views"`) |
| `ClassName` | `string` | Extracted class name (e.g., `"MainPage"`) |
| `RootElementName` | `string` | XAML root element local name (e.g., `"Page"`, `"UserControl"`) |
| `RootElementNamespace` | `string` | XAML root element namespace URI |
| `BaseTypeFullName` | `string` | Resolved base type (e.g., `"Microsoft.UI.Xaml.Controls.Page"`) |

### 3. GenerationConfig (Project-Level Settings)

| Field | Type | Source | Default | Description |
|-------|------|--------|---------|-------------|
| `GlobalEnabled` | `bool` | `build_property.UnoGenerateCodeBehind` | `true` | Project-level enable/disable |

### 4. GeneratedCodeBehind (Output)

The generated source file for a single XAML file.

| Field | Type | Description |
|-------|------|-------------|
| `HintName` | `string` | Source generator hint name (e.g., `"MyApp.Views.MainPage.codebehind.g.cs"`) |
| `SourceText` | `string` | Generated C# source code |

## State Transitions

### Per-XAML-File Decision Flow

```
XAML File Discovered
  │
  ├─ No x:Class attribute? ──────────────── SKIP (FR-006)
  │
  ├─ x:Class malformed? ─────────────────── DIAGNOSTIC + SKIP (Edge case)
  │
  ├─ Per-file UnoGenerateCodeBehind="false"? ── SKIP (FR-008, FR-009)
  │
  ├─ Per-file UnoGenerateCodeBehind="true"? ─── FORCE GENERATE (check class exists)
  │                                           │
  │                                           ├─ Class exists? ── SKIP (FR-005)
  │                                           └─ Class missing? ─ GENERATE
  │
  ├─ No per-file override?
  │   │
  │   ├─ GlobalEnabled = false? ──────────── SKIP (FR-007)
  │   │
  │   └─ GlobalEnabled = true?
  │       │
  │       ├─ Class exists in compilation? ── SKIP (FR-005)
  │       └─ Class missing? ─────────────── GENERATE (FR-001)
  │
  └─ GENERATE:
      Output partial class with:
      - Correct namespace (FR-003)
      - Correct class name (FR-003)
      - Base type from root element (FR-004)
      - Parameterless constructor calling this.InitializeComponent() (FR-002)
      - partial modifier (FR-013)
```

## Generated Code Template

```csharp
// <auto-generated>
// This file was auto-generated by Uno.UI.SourceGenerators.
// Do not edit this file manually.
// </auto-generated>

namespace {Namespace}
{{
    public partial class {ClassName} : {BaseType}
    {{
        public {ClassName}()
        {{
            this.InitializeComponent();
        }}
    }}
}}
```

## Validation Rules

1. **x:Class format**: Must contain at least one `.` separating namespace from class name. If not, emit diagnostic `UXAML0004: Invalid x:Class value '{value}' - must include namespace`.
2. **Duplicate generation**: Generator must not produce output if `Compilation.GetTypeByMetadataName(fullClassName)` returns non-null.
3. **Per-file override precedence**: `UnoGenerateCodeBehind` item metadata always wins over `UnoGenerateCodeBehind` project property.
