# MSBuild Contract: Auto Code-Behind Generation

This feature has no REST/GraphQL APIs. The "contract" is the MSBuild property/metadata interface between the project file and the source generator.

## Properties (Project-Level)

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `UnoGenerateCodeBehind` | `bool` | `true` | Enables/disables auto code-behind generation for the entire project |

## Item Metadata (Per-File)

| Item Type | Metadata | Type | Default | Description |
|-----------|----------|------|---------|-------------|
| `Page` | `GenerateCodeBehind` | `bool?` | (inherit from property) | Per-file override |
| `UnoPage` | `GenerateCodeBehind` | `bool?` | (inherit from property) | Per-file override |
| `ApplicationDefinition` | `GenerateCodeBehind` | `bool?` | (inherit from property) | Per-file override |
| `UnoApplicationDefinition` | `GenerateCodeBehind` | `bool?` | (inherit from property) | Per-file override |

## Generator Output Contract

For each qualifying XAML file, the generator produces a source file with:

```csharp
// <auto-generated/>
namespace {extracted_namespace}
{
    partial class {extracted_classname} : {resolved_base_type}
    {
        public {extracted_classname}()
        {
            this.InitializeComponent();
        }
    }
}
```

## Diagnostics

| ID | Severity | Message |
|----|----------|---------|
| `UNOB0001` | Warning | `Invalid x:Class value '{value}' in '{file}' - must include namespace` |
